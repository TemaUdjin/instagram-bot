# Сессия 2026-05-05 — Старт Desktop App

## Задача
Спроектировать и начать строить нативное Mac-приложение для управления Instagram DM вместо Telegram-бота.

## Что было сделано

### Планирование (сессия 2026-05-04)
- Исследовали существующие аналоги (Inro, ManyChat, IGdm Pro, n8n) — ни один не делает то что нужно
- Выбрали стек: Electron + React + Tailwind
- Составили план в 6 фаз (`plans/2026-05-04-desktop-app.md`)
- Определили дизайн-систему: стиль сайта (#0a0a0a, #c8a96e золото, Geist)
- Добавили идею чата поддержки с сайта как Фазу 6

### Реализация (сессия 2026-05-05)
- Создали структуру проекта `desktop-app/` с electron-vite
- Написали все React компоненты: App, Titlebar, Inbox, Dialog, SuggestionsPanel, Icons
- Настроили дизайн-систему с CSS-переменными (токены сайта)
- Установили все зависимости

### Блокер: macOS 26 несовместима с Electron из npm
- macOS 26.4.1 (Darwin 25) — новая мажорная версия
- Electron из npm не инициализирует нативные модули (`process.type=undefined`)
- `require('electron')` возвращает строку вместо API
- Проверено на Electron 32 и 36, с разными подписями — результат одинаковый

### Решение: переход на Tauri v2
- Tauri использует системный WebKit (всегда совместим с macOS)
- React UI переносится без изменений
- Rust установлен (1.95.0)
- План миграции: `plans/2026-05-05-desktop-app-tauri-migration.md`

## Результат
Частично. UI написан и готов к переносу. Electron не запустился из-за macOS 26. Переходим на Tauri.

## Что можно было сделать лучше
Раньше проверить совместимость Electron с macOS версией. Это сэкономило бы половину сессии.

## Как было → как стало
- Было: Telegram-бот, неудобный интерфейс
- Стало: Готовый React UI, план на Tauri, Rust установлен — следующая сессия сразу к делу
