# Сессия 2026-05-07 — Оптимистичное обновление и обновление inbox

## Задача
1. Оптимистичное обновление после отправки из Claude-панели
2. Обновление статусов в Inbox при отправке сообщения

## Как решал

### 1. Оптимистичное обновление (SuggestionsPanel)
Проблема: `onSelect(msgText)` вызывался только после ответа сервера (`data.ok`).
Исправление: перенёс вызов `onSelect(msgText)` до fetch — сообщение появляется мгновенно.

Бонусный баг: `pendingSentText` в App.tsx был `string`, из-за чего один и тот же текст нельзя отправить дважды подряд (React не видит изменения).
Исправление: тип изменён на `{ text: string; ts: number }`, зависимость в Dialog — по `ts`.

### 2. Обновление Inbox при отправке
Проблема: Inbox опрашивает сервер каждые 20с. После отправки статус `new → replied` обновлялся только через 20с.
Исправление:
- Добавлен `refreshKey?: number` в Inbox (добавлен в зависимости useEffect загрузки)
- Добавлен `onSent?: () => void` в Dialog и SuggestionsPanel
- В App.tsx: `onSent` инкрементирует `inboxRefreshKey` → Inbox сразу перезагружается

## Результат
- Да, решено полностью
- Отправка из поля ввода: уже была оптимистична ✓
- Отправка из Claude-панели: теперь тоже мгновенна ✓
- Inbox обновляет статусы сразу после отправки ✓
